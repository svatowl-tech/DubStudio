**Anime Dub Manager (ADM)** — это передовое кроссплатформенное десктопное приложение для профессиональных студий озвучки, кураторов проектов и фандаб-сообществ. ADM автоматизирует рутину, предлагает инструменты синхронизации и контроля звука, интеллектуального перевода и нарезки субтитров, а также предоставляет детальную аналитику производительности команды.

---

## ✨ Ключевые возможности

### 📂 Управление проектами и релизами
*   **Интеграция с MyAnimeList (MAL):** Мгновенный поиск, автозаполнение карточки тайтла (синопсис, количество серий, постеры, жанры) с возможностью ручной корректировки.
*   **Умное планирование и дедлайны:** Динамический расчет сроков выполнения этапов (Перевод, Озвучка, Тайминг, Сведение) с учетом типа релиза (Онгоинг / Оффгоинг) и формата озвучки (Закадр / Рекаст / Редаб).
*   **Гармоничный интерфейс списков:** Удобное отслеживание статуса каждой серии, назначенных даберов, таймеров и переводчиков с визуальной цветовой индикацией дедлайнов.

### 📝 Умная работа с субтитрами (ASS Subtitle Suite)
*   **ИИ-Переводчик (OpenRouter API):** Интегрированный переводчик субтитров с поддержкой современных LLM (Gemini, Claude, GPT). Интеллектуальный перевод учитывает выбранный в настройках жанр аниме, характер персонажей и контекст сцен.
*   **Автоматическое разделение реплик:** Утилита мгновенно нарезает общий ASS-файл на индивидуальные текстовые скрипты для каждого дабера, исключая ручную рутину.
*   **Редактор реплик:** Удобная таблица реплик с фильтрацией по персонажам, интерактивным поиском и моментальным редактированием тайминга прямо в приложении.

### 🎧 Интерактивная Панель QA (Контроль качества)
*   **Мультитрековый плеер в реальном времени:** Позволяет кураторам прослушивать присланные аудиодорожки даберов поверх исходного видеофайла без предварительного перекодирования.
*   **Шкала таймлайна с Wavesurfer.js:** Отображение волновой диаграммы аудио для точного контроля попадания реплик в липсинк и тайминг.
*   **Конструктор правок (Фиксов):** Маркировка проблемных мест на временной шкале и формирование детализированных отчетов для даберов в один клик.

### 📊 Интеллектуальная Аналитика и Статистика (StatsPanel)
*   **Панель эффективности команды:** Детальный анализ активности даберов, звукорежиссеров и кураторов.
*   **Динамика производительности:** Наглядные интерактивные графики (Recharts) с распределением по ролям, среднему времени сдачи материалов, проценту просроченных дедлайнов и объемам выполненной работы.
*   **Рейтинги и кубки (Leaderboards):** Мотивационные топы участников команды с распределением наград по результатам работы.

### 🎨 Генератор Обложек (CoverGenerator)
*   **Шаблоны профессионального уровня:** Наложение логотипа на кадр из видео, настройка разделительного узора, подложки и стиля под ваше сообщество.
*   **Гибкое форматирование текста:** Поддерживает выбор из популярных шрифтов (Russo One, Inter, JetBrains Mono и др.), изменение размеров, обводки, мягких теней и многострочный перенос заголовков.
*   **Интегрированное позиционирование:** Полный контроль над сдвигом текста по координатам X/Y и расстоянием между строками в реальном времени с автосохранением шаблона в настройках проекта.

## 🛠 Технологический Стек

*   **Runtime & API:** Electron (Main & Renderer IPC безопасности)
*   **Фронтенд:** React 19, TypeScript, Vite, Tailwind CSS v4.0
*   **Анимации:** Motion (ранее framer-motion)
*   **База данных:** Локальное защищенное JSON-хранилище проектов и команды с автоматическим бэкапом
*   **Обработка медиа:** FFmpeg (с поддержкой `fluent-ffmpeg`), Wavesurfer.js
*   **Машинное обучение:** Локальные ONNX-модели (PyAnnote для диаризации речи, M2M100 для перевода, Whisper для ASR)

## 🚀 Быстрый старт

### Системные требования
*   Node.js v18 или v20
*   Git

### Пошаговая установка
1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/SvatOwl/anime-dub-manager.git
    cd anime-dub-manager
    ```

2.  **Установите npm-зависимости:**
    ```bash
    npm install
    ```

3.  **Запустите режим разработки:**
    ```bash
    npm run electron:dev
    ```

## 📦 Сборка и Релиз Готового Приложения

Для компиляции приложения под вашу текущую ОС выполните команду:
```bash

```
Готовые установщики будут сгенерированы в директории `dist/electron/` (`.exe` для Windows, `.dmg` для macOS, `.AppImage` / `.deb` для Linux).

## 🤖 Автоматическая сборка в Облаке (GitHub Actions)

В репозитории настроен высокоэффективный CI/CD-конвейер через GitHub Actions (`.github/workflows/release.yml`), автоматизирующий создание релизов.

### Преимущества нашей сборки в облаке:
1.  **Кроссплатформенность:** Сборка ведется параллельно на трех виртуальных машинах: `windows-latest`, `macos-latest` и `ubuntu-latest`.
2.  **Интеллектуальное кэширование:** FFmpeg-бинарники и ресурсоемкие локальные ИИ-модели кэшируются между сборками (`actions/cache`). Это сокращает время выполнения пайплайна с 15 минут до 2 минут.
3.  **Авто-публикация:** Интеграция `electron-builder` автоматически загружает сгенерированные файлы к релизу в статусе *Draft* на GitHub.

### 📜 Как активировать автоматический релиз на GitHub (Инструкция)

Чтобы настроить автоматическую сборку на вашем репозитории при выпуске новой версии:

1.  **Создайте репозиторий на GitHub** и отправьте туда ваш код:
    ```bash
    git remote add origin https://github.com/ВАШ_ЛОГИН/anime-dub-manager.git
    git branch -M main
    git push -u origin main
    ```

2.  **Настройте права GITHUB_TOKEN (Критически важно!):**
    По умолчанию GitHub Actions не имеет прав на создание релизов в вашем репозитории.
    *   Перейдите в ваш репозиторий на GitHub.
    *   Откройте **Settings** ➔ **Actions** ➔ **General**.
    *   Прокрутите вниз до раздела **Workflow permissions**.
    *   Выберите **Read and write permissions** (Разрешить чтение и запись).
    *   Нажмите **Save**.

3.  **Запуск релиза через Тегирование:**
    Когда вы готовы выпустить новую версию (например, `1.1.0`):
    *   Обновите версию в `package.json` на `"version": "1.1.0"`.
    *   Зафиксируйте изменения и создайте Git-тег:
        ```bash
        git add package.json
        git commit -m "chore: bump version to 1.1.0"
        git push origin main
        
        git tag v1.1.0
        git push origin v1.1.0
        ```
    *   Перейдите во вкладку **Actions** в веб-панели GitHub. Вы увидите запущенный процесс сборки.
    *   После успешного завершения сборки во вкладке **Releases** появится черновик релиза (**Draft Release**), содержащий все файлы:
        *   `Anime-Dub-Manager-Setup-1.1.0.exe` (Инсталлятор Windows NSIS)
        *   `Anime-Dub-Manager-Portable-1.1.0.exe` (Портативная версия Windows)
        *   `Anime-Dub-Manager-1.1.0.dmg` (Установщик macOS)
        *   `Anime-Dub-Manager-1.1.0.AppImage` (Linux)
    *   Отредактируйте описание релиза и нажмите **Publish Release**!
